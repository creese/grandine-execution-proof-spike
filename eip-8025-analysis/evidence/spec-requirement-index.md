## Unaffected client responsibilities inferred from the sources

**Cross-client inference:** beacon-state schema, block schema, proposer/attester duties, slashing conditions, and consensus fork-choice weighting are unchanged because proofs remain out-of-block and supplementary. Nodes with both modes disabled should not subscribe, advertise, request, prove, verify, or retain proof data. Phase 2 must verify whether Lighthouse preserves each boundary rather than treating this inference as a Lighthouse fact.
